# 원재 연산앱

초등학교 3학년 아동이 3자리 덧셈/뺄셈의 자리올림·자리내림 유형을 **분리해서 반복 연습**할 수 있도록 만든 모바일 우선 웹 앱 MVP입니다.

## 이번 구현 범위
- 홈
- 유형 선택
- 문제 풀이
- 결과
- 오답만 다시 풀기
- 틀린 유형만 다시 / 가장 약한 유형 새 문제 10개
- 세로셈 힌트 3단계
- 최근 학습 기록 화면
- 유형별 정확도 로컬 저장
- 오늘 추천 연습 / 약한 유형 추천 기본 구조
- 보호자용 최소 요약 카드

## 2차 정리 반영
- 홈, 유형 선택, 문제 풀이, 결과 화면 문구를 더 짧고 쉬운 말로 정리했습니다.
- 덧셈과 뺄셈 카드를 그룹 색과 규칙 칩으로 나눠 헷갈림을 줄였습니다.
- 문제 화면에서 현재 유형, 문제 번호, 식, 자리값 보드가 더 바로 보이도록 구조를 다듬었습니다.
- 피드백 문구를 짧게 줄이고 반복해서 읽기 쉬운 표현으로 맞췄습니다.

## 핵심 특징
- 5개 실제 유형을 분리 출제하거나, `전체 유형 랜덤` 모드로 섞어서 출제할 수 있습니다.
- 문제 생성은 `생성 -> 분석 -> 유형 검증` 순서로 안전하게 통과한 문제만 사용합니다.
- 문제 화면은 자리값이 보이도록 `천 / 백 / 십 / 일` 그리드를 사용합니다.
- `세로셈 힌트`를 `숨김 / 먼저 / 직접` 3단계로 바꿨습니다.
- `먼저` 단계는 아이가 어느 자리부터 볼지 한 자리만 먼저 보여주고, `직접` 단계는 올림/내림이 생기는 자리를 더 직접적으로 보여줍니다.
- `3자리 덧셈`처럼 올림이 없는 기본형은 `먼저`에서 일 자리부터 보게 하고, `직접`에서는 자리 맞춤 흐름을 짧게 알려줍니다.
- 세션은 기본 10문제이며, 결과에서 `틀린 문제 다시`, `가장 약한 유형 새 문제 10개`, `틀린 유형만 다시`로 이어서 복습할 수 있습니다.
- `localStorage`에 누적 정확도와 최근 학습 기록을 저장합니다.

## 오답 복습 강화
- 오답 힌트는 `일 자리 올림 다시.`, `십 자리 내림 다시.`처럼 더 짧고 구체적인 자리 힌트로 보여줍니다.
- 문제 보드에서도 같은 자리 열을 바로 다시 보도록 연한 강조와 오답 강조를 함께 보여줍니다.
- 결과 화면에서 이번 세션의 약한 유형 1~2개를 먼저 보여줍니다.
- 혼합 모드에서도 틀린 실제 유형만 골라 새 10문제를 다시 만들 수 있습니다.
- 단일 유형 모드에서는 같은 유형 새 문제 10개를 바로 다시 시작할 수 있습니다.

## 단계형 힌트와 결과 추천
- 문제 화면 상단에서 `숨김 / 먼저 / 직접` 버튼으로 힌트 단계를 바로 바꿀 수 있습니다.
- `먼저`는 한 자리만 짚어 주고, `직접`은 실제 올림/내림 자리 전체를 보여 줘 차이를 바로 이해할 수 있게 했습니다.
- 결과 화면에는 이번 세션을 바탕으로 한 줄 추천 문구와 연결된 대표 CTA를 함께 보여줍니다.
- 단일 유형에서 틀리면 같은 유형 새 문제를 먼저 권하고, 혼합 모드에서 여러 유형이 흔들리면 `틀린 유형만 다시`를 먼저 권합니다.
- 단일 유형을 모두 맞히면 다음 단계 유형으로 올려 보도록 추천할 수 있습니다.

## 보호자용 최소 요약
- 홈에서 최근 연습 횟수, 최근 학습 기록 미리보기, 오늘 추천 유형을 카드로 보여줍니다.
- 누적 기록이 있는 유형만 골라 간단한 정답률 요약을 한 카드 안에 보여줍니다.
- 가장 약한 유형 1~2개를 따로 보여 다음 연습 선택을 빠르게 도와줍니다.

## 최근 학습 기록 화면
- 홈의 `최근 학습 기록` 버튼으로 최근 세션 목록 화면으로 이동할 수 있습니다.
- 각 세션 카드에는 세션 모드, 정확도, 완료 시간, 맞은 개수, 실제 유형 분포를 함께 보여줍니다.
- `혼합 모드`, `틀린 문제 다시`, `틀린 유형만 다시`, `새 문제` 같은 레이블로 최근 흐름을 구분합니다.
- 홈의 보호자용 최소 요약은 누적 정확도 중심, 최근 학습 기록 화면은 최근 세션 흐름 중심으로 역할을 나눴습니다.

## 문제 유형
- 전체 유형 랜덤: 10문제 동안 5개 실제 유형이 섞여 나오는 혼합 모드
- 3자리 덧셈: 현재 MVP에서는 `받아올림 없는 기본형`
- 받아올림 2번
- 받아올림 3번
- 받아내림 1번
- 받아내림 2번

## 랜덤 모드 동작 방식
- 세션 모드 id는 `mixed-all`로 저장합니다.
- 각 문제는 생성 시점에 실제 `problem.typeId`를 따로 가집니다.
- 랜덤 세션도 문제 생성 규칙은 느슨해지지 않습니다. 각 문제는 자기 `problem.typeId` 기준으로 기존 검증을 통과한 뒤에만 출제됩니다.
- 문제 화면에서는 세션 모드와 별개로 현재 문제의 실제 유형을 함께 보여줍니다.
- 결과 화면과 오답 다시 풀기에서도 각 문제의 실제 유형 정보를 그대로 사용합니다.
- 혼합 세션 결과에서는 틀린 실제 유형만 다시 모아 새 문제 10개를 만들 수 있습니다.

## 기록 저장 메모
- 누적 `stats`는 계속 실제 5개 유형 기준으로 쌓입니다.
- 최근 `history`에는 세션 모드 id(`sessionTypeId`), 세션 표시 이름, 유형 분포, 틀린 유형 분포가 함께 저장됩니다.
- 최근 기록 화면은 같은 `history` 데이터를 읽어 세션 모드 레이블, 정확도, 시간, 실제 유형 분포를 카드 형태로 보여줍니다.
- 따라서 혼합 모드를 추가해도 기존 `localStorage` 구조를 크게 바꾸지 않고 읽을 수 있습니다.

## 실행 방법
### 테스트
```bash
npm test
```

### 정적 빌드
```bash
npm run build
```

### 로컬 서버
```bash
npm start
```

샌드박스 환경에서는 포트 바인딩이 막혀 `npm start`가 실패할 수 있습니다. 일반 로컬 환경에서는 `http://127.0.0.1:4173`에서 확인하도록 구성했습니다.

## 배포
GitHub Actions로 GitHub Pages 배포를 하도록 `.github/workflows/deploy-pages.yml`를 추가했습니다.

### Pages 활성화
GitHub 저장소 설정에서 아래처럼 맞추면 됩니다.
- **Settings > Pages**
- **Build and deployment > Source**: `GitHub Actions`

그다음 `main` 브랜치에 push하면 자동으로:
1. `npm test`
2. `npm run build`
3. `dist/` 업로드
4. GitHub Pages 배포

형태로 배포됩니다.

## 검증 포인트
- 각 유형 생성 문제가 실제 규칙과 일치하는지 `tests/problem-engine.test.js`에서 검사합니다.
- `3자리 덧셈` 설명이 `받아올림 없는 기본형`과 맞는지 메타데이터 테스트로 함께 확인합니다.
- `전체 유형 랜덤` 세션에서도 각 문제가 자기 실제 유형 규칙을 만족하는지 함께 검사합니다.
- `틀린 유형만 다시` 세션이 요청된 실제 유형만 다시 쓰는지도 함께 검사합니다.
- `tests/learning-support.test.js`에서 힌트 단계 노출 규칙과 결과 추천 문구 분기를 함께 검사합니다.
- 샘플 문제를 직접 생성해 carry/borrow 횟수도 함께 확인할 수 있습니다.
