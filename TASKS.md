## 1차 구현 메모

- Stitch MCP 검토 내용은 `docs/STITCH-REVIEW.md`에 정리했다.

## 2차 정리 메모
- 홈, 유형 선택, 문제 풀이, 결과 화면 문구를 초3 기준으로 더 짧게 다듬었다.
- 유형 카드에 `덧셈 / 뺄셈` 그룹 칩과 `올림 없음 / 일, 십 / 일, 십, 백` 규칙 칩을 추가해 시각 구분을 강화했다.
- 문제 화면에서 현재 유형 카드, 문제 번호 카드, 식 표시, 자리값 보드 대비를 더 크게 만들었다.
- `3자리 덧셈`이 현재는 `받아올림 없는 기본형`이라는 점을 UI와 README에 명확히 적었다.
- `.gitignore`에 `.omx/`를 추가해 작업 산출물이 추적되지 않도록 정리했다.

## 3차 확장 메모
- `전체 유형 랜덤` 세션 모드를 추가했다.
- 세션 모드 id(`mixed-all`)와 실제 문제 유형 id(`problem.typeId`)를 분리해 혼합 세션에서도 기존 유형 검증을 그대로 유지했다.
- 결과 화면에서 전체 점수와 함께 이번에 나온 실제 유형, 틀린 실제 유형, 틀린 문제의 실제 유형을 같이 보여주도록 정리했다.
- 오답 다시 풀기에서도 각 문제의 실제 유형 정보를 그대로 유지하도록 세션 재생성 흐름을 보강했다.

## 4차 확장 메모
- 결과 화면에 `가장 약한 유형 새 문제 10개`와 `틀린 유형만 다시` CTA를 추가해 오답 복습 경로를 넓혔다.
- 혼합 모드 결과에서도 틀린 실제 유형만 모아 새 10문제를 다시 생성할 수 있게 세션 생성 함수를 확장했다.
- 오답 힌트를 `일 자리 올림 다시.` 같은 짧은 자리 중심 문구로 더 구체화했다.
- 홈 화면에 최근 연습 횟수, 가장 약한 유형 1~2개, 유형별 정답률 요약, 오늘 추천 유형을 카드 몇 개 수준으로 추가했다.

## 5차 확장 메모
- 문제 풀이 화면에 `세로셈 힌트 보기 / 숨기기` 토글을 추가했다.
- 힌트를 켜면 자리값 보드에서 관련 자리 열을 연하게 강조하고, 오답 제출 뒤에는 같은 자리를 더 강하게 표시하도록 정리했다.
- 홈에서 들어갈 수 있는 `최근 학습 기록` 전용 화면을 추가해 세션 모드, 정확도, 시간, 실제 유형 분포를 최근 순으로 보이게 했다.
- 홈 카드에서는 최근 기록 미리보기만 보여주고, 보호자용 최소 요약은 누적 정확도 중심 역할로 분리했다.

## 6차 확장 메모
- 문제 풀이 화면의 힌트를 `숨김 / 먼저 / 직접` 3단계로 바꿨다.
- `먼저`는 시작할 자리 하나만, `직접`은 실제 올림/내림 자리를 더 직접적으로 보여주도록 보드 강조와 문구를 나눴다.
- 결과 화면에 이번 세션 기준 한 줄 추천 카드와 연결된 대표 CTA를 추가했다.
- 단일 유형 오답, 혼합 모드 오답, 만점 세션을 나눠 다음 추천 연습 문구와 버튼이 자연스럽게 이어지도록 정리했다.
- 힌트 단계와 결과 추천 규칙은 `tests/learning-support.test.js`로 고정했다.
