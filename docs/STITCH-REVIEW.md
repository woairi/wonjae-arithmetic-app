# Stitch 검토 메모

## 목적
원재 연산앱 구현 전에 Stitch MCP를 사용해 모바일 학습 앱 화면 구조를 먼저 검토하고, 실제 구현 가능한 UI 원칙으로 정리한다.

## 이번 구현에서 사용한 Stitch MCP 작업
- `list_projects`
- `get_project`
- `get_screen` x 4

## 검토한 Stitch 화면
작업형 생성 호출은 이 환경에서 반복 취소되어, 기존 소유 모바일 교육용 Stitch 프로젝트를 검토 대상으로 사용했다.

프로젝트:
- `projects/11649734933065250022`

화면:
1. `Home & Mission`
2. `Learning Card`
3. `My Progress`
4. `Hanja Garden Learning App`

## 반영한 UI 원칙
1. 홈은 긴 설명보다 큰 시작 버튼과 추천 카드가 먼저 보여야 한다.
2. 카드와 버튼은 둥글고 여백이 넉넉해야 한다.
3. 숫자 화면은 작은 텍스트보다 큰 숫자와 단순한 입력 흐름이 우선이다.
4. 결과 화면은 분석보다 다음 행동 버튼이 먼저 보여야 한다.
5. 구분선 대신 배경 톤 차이와 카드 레이어로 구조를 만든다.

## 실제 구현으로 옮기며 줄인 부분
- 과한 장식 일러스트는 제거
- 화면당 CTA는 1~2개로 축소
- 문제 화면은 학습 안정성을 위해 자리값 그리드 중심으로 단순화
- 결과 화면은 오답 다시 풀기, 같은 유형 다시 하기, 다른 유형 선택만 유지

## 2차 반영 메모
- 홈은 큰 시작 버튼과 추천 카드 중심 흐름을 유지하되 문구를 더 짧게 정리했다.
- 유형 선택은 `덧셈 / 뺄셈` 그룹을 색과 칩으로 나눠 구조 반영을 더 또렷하게 만들었다.
- 문제 화면은 `현재 유형 카드 + 문제 번호 카드 + 식 + 자리값 보드` 순서로 위계를 다시 세웠다.
- 결과 화면은 점수 설명을 줄이고 다시 할 행동 버튼이 먼저 보이도록 다듬었다.
