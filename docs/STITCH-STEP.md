# Stitch UI 디자인 단계

## 목적
Codex 구현 전에 원재 연산앱의 핵심 화면 구조를 Stitch MCP로 먼저 설계한다.

## 먼저 만들 화면
1. 홈
2. 유형 선택
3. 문제 풀이
4. 결과

## 검토 포인트
- 숫자가 충분히 크게 보이는가
- 유형 카드가 헷갈리지 않는가
- 문제 풀이 흐름이 단순한가
- 결과 화면에서 다음 행동이 분명한가
- 초3 아이가 혼자 눌러도 부담 없을 정도로 단순한가

## 다음 단계
- Stitch 화면 구조 검토
- 필요한 경우 UI-SPEC 보정
- 그 후 Codex 구현 시작
