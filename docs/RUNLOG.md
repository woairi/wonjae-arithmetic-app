# RUNLOG — 원재 연산앱

### Loop 1
- 목표: 새 프로젝트를 만들고, PRD / 문제 생성 규칙 / MVP / TASKS / Codex 1차 프롬프트를 준비한다.
- Codex CLI에 넘긴 작업: 없음 (준비 단계)
- 결과 요약: 초3 대상 연산 훈련 앱으로 프로젝트를 정의했고, 유형 분리 학습에 맞는 PRD와 문제 생성 규칙 문서를 작성했다. 이후 MVP 범위를 정리하고 TASKS 및 Codex 1차 구현 프롬프트를 작성했다.
- 통과/실패 판단: 통과
- 추가 지시 사항: 다음 루프에서 Codex CLI에 1차 구현 작업을 실제로 넘긴다.
- 남은 리스크: 문제 생성 엔진이 이 프로젝트의 핵심이므로, 구현 검토 시 UI보다 규칙 정확성을 우선 확인해야 한다.

### Loop 2
- 목표: Stitch 선행 검토를 포함한 1차 MVP 구현을 진행한다.
- Codex CLI에 넘긴 작업: 홈/유형 선택/문제 풀이/결과/오답 복습/로컬 저장/문제 생성 엔진 구현.
- 결과 요약: 정적 모바일 웹 앱 골격과 10문제 세션 흐름, 오답 다시 풀기, 로컬 저장, 문제 생성/분석/검증 엔진, 테스트 및 빌드 스크립트가 구현되었다. Stitch 검토 메모와 README/TASKS/CHANGELOG도 함께 갱신되었다.
- 통과/실패 판단: 조건부 통과
- 추가 지시 사항: 다음 루프에서 아이 친화 문구, 유형 카드 구분, 문제 풀이 화면 읽힘, 설명 정밀화, 프로젝트 정리를 보강한다.
- 남은 리스크: Stitch 반영은 간접적이며, 초3 기준 UI 문구/시각 구분은 한 번 더 다듬을 필요가 있다.

### Loop 3
- 목표: 기존 5개 유형 외에 `전체 유형 랜덤` 모드를 추가하고, 혼합 세션에서도 실제 유형 정보를 끝까지 유지한다.
- Codex CLI에 넘긴 작업: 혼합 세션 모드 추가, 세션 모드 id와 실제 문제 type id 분리, 결과/오답 화면 보강, 저장 구조 확장, 문서와 테스트 갱신.
- 결과 요약: `mixed-all` 세션 모드를 추가했고, 10문제 세션에서 5개 실제 유형을 섞어 출제하도록 구현했다. 각 문제는 생성 시점부터 실제 `problem.typeId`를 유지하며, 문제 화면·결과 화면·오답 다시 풀기·`localStorage` 기록에 그 정보가 이어지도록 정리했다. 혼합 세션 검증 테스트와 빌드도 통과했다.
- 통과/실패 판단: 통과
- 추가 지시 사항: 사용자 최종 검토에서 혼합 모드 문구와 결과 화면의 실제 유형 표시가 기대와 맞는지 확인한다.
- 남은 리스크: 혼합 세션이 실제 유형별 누적 통계에 반영되므로, 향후 통계 화면을 따로 만들면 `sessions` 의미를 "전용 연습 횟수"로 볼지 "등장 세션 수"로 볼지 다시 정의할 필요가 있다.

### Loop 4
- 목표: 오답 복습을 더 학습 효과 있게 만들고, 홈 화면에 보호자용 최소 요약을 추가한다.
- Codex CLI에 넘긴 작업: 오답 유형 기반 새 문제 세션 추가, 결과 화면 약한 유형 강조, 짧고 구체적인 오답 힌트, 보호자용 최소 요약 카드, 테스트와 문서 갱신.
- 결과 요약: 결과 화면에 `가장 약한 유형 새 문제 10개`와 `틀린 유형만 다시`를 추가했고, 혼합 모드에서도 틀린 실제 유형만 골라 다시 연습할 수 있게 했다. 홈 화면에는 최근 연습 횟수, 가장 약한 유형 1~2개, 유형별 정답률 요약, 오늘 추천 유형을 카드 몇 개 수준으로 추가했다. 새 오답 힌트와 저장 구조 확장 테스트, 빌드도 통과했다.
- 통과/실패 판단: 통과
- 추가 지시 사항: 실제 모바일 화면에서 결과 CTA 밀도와 보호자 요약 가독성을 한 번 더 확인한다.
- 남은 리스크: 보호자 요약은 누적 로컬 기록 기준이라 기록이 적을 때 추천 정확도가 낮을 수 있고, 혼합 세션의 "같은 모드 새 문제 10개"와 "틀린 유형만 다시" 문구 차이는 실제 사용자에게 짧은 설명이 더 필요할 수 있다.

### Loop 5
- 목표: 문제 화면에 자리값 힌트 토글과 관련 자리 강조를 추가하고, 홈에서 갈 수 있는 최근 학습 기록 화면을 만든다.
- Codex CLI에 넘긴 작업: 세로셈 힌트 on/off, 문제 보드 자리 강조, 오답 시 관련 자리 시각화, 최근 학습 기록 화면 추가, 홈 연결, 테스트/빌드/문서 갱신.
- 결과 요약: 문제 풀이 화면에 `세로셈 힌트 보기 / 숨기기` 토글을 추가했고, 힌트를 켜면 자리값 보드에서 관련 자리 열을 연하게 강조하도록 했다. 오답 제출 뒤에는 같은 자리 열을 한 단계 더 강하게 표시하고 짧은 자리 포인트 문구를 함께 보여주도록 정리했다. 홈에서는 최근 기록 미리보기와 `최근 학습 기록` 진입 버튼을 추가했고, 전용 기록 화면에서는 세션 모드, 정확도, 완료 시간, 실제 유형 분포를 최근 순 카드 목록으로 볼 수 있게 했다. 테스트와 빌드도 통과했다.
- 통과/실패 판단: 통과
- 추가 지시 사항: 실제 모바일 브라우저에서 힌트 강조 강도와 최근 기록 카드 길이가 초3 기준으로 과하지 않은지 확인한다.
- 남은 리스크: 힌트 강조는 현재 문제의 실제 자리 포인트를 직접 보여주는 구조라 난이도를 낮추는 대신 정답 추론 보조가 강해질 수 있고, 최근 기록 화면은 로컬 저장 20개 제한 안에서만 보인다.

### Loop 6
- 목표: 자리값 힌트를 단계형으로 바꾸고, 결과 화면에서 다음 추천 연습과 CTA 연결을 더 자연스럽게 만든다.
- Codex CLI에 넘긴 작업: 세로셈 힌트 3단계, 보드 강조 규칙 분리, 결과 추천 문구/대표 CTA 분기, 테스트/빌드/문서 갱신.
- 결과 요약: 문제 풀이 화면의 힌트를 `숨김 / 먼저 / 직접` 3단계로 확장했고, `먼저` 단계는 시작할 자리 하나만, `직접` 단계는 실제 올림/내림 자리 전체를 더 직접적으로 보여주도록 정리했다. 결과 화면에는 이번 세션 기준 한 줄 추천 카드와 연결된 대표 CTA를 추가했고, 단일 유형 오답/혼합 모드 오답/만점 세션에서 각각 자연스럽게 이어지도록 분기했다. `tests/learning-support.test.js`, `npm test`, `npm run build`도 모두 통과했다.
- 통과/실패 판단: 통과
- 추가 지시 사항: 실제 모바일 브라우저에서 `숨김 / 먼저 / 직접` 버튼 길이와 추천 카드 버튼 우선순위를 한 번 더 확인한다.
- 남은 리스크: 현재 힌트 2단계는 실제 올림/내림 자리를 직접 보여 주므로 일부 아이에게는 도움 강도가 크게 느껴질 수 있고, 결과 추천은 세션 규칙 기반이라 장기 누적 학습 패턴까지는 아직 반영하지 않는다.

### Loop 7
- 목표: 최근 7일 학습 흐름을 한눈에 요약하고, 다음 추천 유형을 더 자연스럽고 똑똑하게 만든다.
- Codex CLI에 넘긴 작업: 최근 7일 요약 카드 추가, 최근 7일 집계 함수 구현, 홈/결과 추천 로직 고도화, 추천 이유 표시, 테스트/빌드/문서 갱신.
- 결과 요약: 홈과 최근 학습 기록 화면에 최근 7일 연습 횟수, 가장 많이 푼 유형, 가장 많이 틀린 유형, 평균 정확도를 보여주는 카드형 요약을 추가했다. 추천 로직은 최근 7일 약한 유형, 최근 같은 유형의 2회 이상 연속 성공, 어려운 유형에서 연속 저정확도일 때 한 단계 쉬운 복귀, 여러 유형을 고르게 잘 푼 주간의 혼합 모드 추천까지 함께 보도록 확장했다. 홈과 결과 화면의 추천 카드에는 짧은 `이유:` 문구도 붙였다. `tests/storage.test.js`, `tests/learning-support.test.js`, `npm test`, `npm run build`가 모두 통과했다.
- 통과/실패 판단: 통과
- 추가 지시 사항: 실제 모바일 브라우저에서 홈 카드 길이와 최근 학습 기록 화면 상단 카드 밀도를 한 번 더 확인한다.
- 남은 리스크: 최근 7일 요약은 로컬 `history` 최대 20개 안에서만 계산되므로 매우 짧은 세션을 하루에 많이 반복하면 일부 오래된 주간 세션이 밀릴 수 있고, 추천은 규칙 기반이라 학습 의도 변화까지는 아직 직접 묻지 않는다.
