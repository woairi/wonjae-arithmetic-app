# RUNLOG — 원재 연산앱

### Loop 1
- 목표: 새 프로젝트를 만들고, PRD / 문제 생성 규칙 / MVP / TASKS / Codex 1차 프롬프트를 준비한다.
- Codex CLI에 넘긴 작업: 없음 (준비 단계)
- 결과 요약: 초3 대상 연산 훈련 앱으로 프로젝트를 정의했고, 유형 분리 학습에 맞는 PRD와 문제 생성 규칙 문서를 작성했다. 이후 MVP 범위를 정리하고 TASKS 및 Codex 1차 구현 프롬프트를 작성했다.
- 통과/실패 판단: 통과
- 추가 지시 사항: 다음 루프에서 Codex CLI에 1차 구현 작업을 실제로 넘긴다.
- 남은 리스크: 문제 생성 엔진이 이 프로젝트의 핵심이므로, 구현 검토 시 UI보다 규칙 정확성을 우선 확인해야 한다.

### Loop 2
- 목표: Stitch 선행 검토를 포함한 1차 MVP 구현을 진행한다.
- Codex CLI에 넘긴 작업: 홈/유형 선택/문제 풀이/결과/오답 복습/로컬 저장/문제 생성 엔진 구현.
- 결과 요약: 정적 모바일 웹 앱 골격과 10문제 세션 흐름, 오답 다시 풀기, 로컬 저장, 문제 생성/분석/검증 엔진, 테스트 및 빌드 스크립트가 구현되었다. Stitch 검토 메모와 README/TASKS/CHANGELOG도 함께 갱신되었다.
- 통과/실패 판단: 조건부 통과
- 추가 지시 사항: 다음 루프에서 아이 친화 문구, 유형 카드 구분, 문제 풀이 화면 읽힘, 설명 정밀화, 프로젝트 정리를 보강한다.
- 남은 리스크: Stitch 반영은 간접적이며, 초3 기준 UI 문구/시각 구분은 한 번 더 다듬을 필요가 있다.
