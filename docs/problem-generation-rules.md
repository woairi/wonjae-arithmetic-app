# 문제 생성 규칙

이 문서는 원재 연산앱의 핵심인 **문제 생성 규칙**을 정의한다.
핵심 원칙은 “유형 이름과 실제 문제 구조가 정확히 일치해야 한다”이다.

## 1. 유형 목록

### 덧셈
1. 3자리 덧셈 기본
2. 받아올림 2번
3. 받아올림 3번

### 뺄셈
4. 받아내림 1번
5. 받아내림 2번

---

## 2. 덧셈 규칙

### A. 3자리 덧셈 기본
- 세 자리 수 + 세 자리 수
- 받아올림이 없거나, MVP에서는 받아올림이 섞이지 않게 별도 정의하는 것이 안전함
- 초기 MVP에서는 “받아올림 없음” 버전으로 두는 것을 권장

예:
- 123 + 245
- 314 + 262

### B. 받아올림 2번
- 일의 자리와 십의 자리에서 받아올림이 발생하고, 백의 자리에서는 추가 받아올림이 발생하지 않도록 생성
- 또는 십의 자리와 백의 자리에서 받아올림이 발생하는 패턴은 별도 고려 가능하지만, 초기 MVP는 한 패턴으로 고정하는 것이 안전

권장 초기 패턴:
- 일의 자리 합 >= 10
- 십의 자리 합 + 이전 받아올림 >= 10
- 백의 자리 합 + 이전 받아올림 < 10

예:
- 278 + 356
  - 일의 자리: 8+6=14 (1차)
  - 십의 자리: 7+5+1=13 (2차)
  - 백의 자리: 2+3+1=6

### C. 받아올림 3번
- 일의 자리, 십의 자리, 백의 자리 모두 연속적으로 받아올림 발생

규칙:
- 일의 자리 합 >= 10
- 십의 자리 합 + 이전 받아올림 >= 10
- 백의 자리 합 + 이전 받아올림 >= 10

예:
- 578 + 456
  - 일의 자리: 8+6=14
  - 십의 자리: 7+5+1=13
  - 백의 자리: 5+4+1=10

---

## 3. 뺄셈 규칙

### A. 받아내림 1번
- 한 자리에서만 받아내림이 발생하도록 생성
- 초기 MVP에서는 **일의 자리에서만 받아내림 발생** 패턴으로 고정하는 것이 안전

권장 초기 패턴:
- 일의 자리: 윗수 < 아랫수 → 받아내림 1번
- 십의 자리: 받아내림 반영 후도 계산 가능
- 추가 받아내림은 발생하지 않음

예:
- 352 - 146
  - 일의 자리: 2 < 6 → 받아내림
  - 십의 자리: 5에서 1을 빌려줘도 4 >= 4
  - 백의 자리는 추가 받아내림 없음

### B. 받아내림 2번
- 일의 자리와 십의 자리에서 연속으로 받아내림 발생

규칙:
- 일의 자리: 윗수 < 아랫수
- 십의 자리: 이전 받아내림 반영 후도 윗수 < 아랫수
- 백의 자리에서는 추가 받아내림 없음

예:
- 402 - 178
  - 일의 자리: 2 < 8 → 받아내림
  - 십의 자리: 0에서 하나 빌려 썼기 때문에 더 작아져 다시 받아내림

---

## 4. MVP 생성 원칙
1. 한 유형 안에 패턴을 너무 많이 섞지 않는다.
2. 처음에는 **대표 패턴 1개씩**만 정확하게 생성한다.
3. 유형 판정 함수와 문제 생성 함수를 분리한다.
4. 생성 후 반드시 규칙 검증 함수를 통과한 문제만 출제한다.

---

## 5. 구현 권장 방식
- `generateProblem(type)`
- `analyzeProblem(problem)`
- `isValidForType(problem, type)`

권장 흐름:
1. 후보 문제 생성
2. 분석 함수로 실제 자리올림/자리내림 횟수 계산
3. 원하는 유형과 일치하면 채택
4. 아니면 폐기 후 재생성

---

## 6. 주의사항
- “받아올림 2번”인데 실제론 3번 발생하는 문제를 내면 안 된다.
- “받아내림 1번”인데 실제론 2번 발생하는 문제를 내면 안 된다.
- 문제 생성 로직의 정확성이 UI보다 우선이다.
- 처음에는 패턴을 단순하게 고정하고, 나중에 변형 패턴을 늘리는 방식이 안전하다.
